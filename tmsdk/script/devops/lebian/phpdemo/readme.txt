为保证数据安全，请求乐变API须对数据进行加密（登录除外），使用RSA加密方式
我们提供了一份API文档（lbapi.doc）和一套简单的demo，希望两者结合能有助于您更好地理解和使用乐变系统

demo中提供了两种方式供您把一个包上传到乐变后台，请根据需要，选择其中一种。
下文讲的接口，实为接口中action的值

情景一：
任意一个包，请求batchUpload接口，如果该渠道不存在，尝试创建一个渠道。如果该渠道存在，尝试更新渠道上的版本。
这种情形，请参考demo1.php
注1.如果可行，将会使用默认设置，中间不会有交互。
注2.如果同一个渠道有多个包，请按versioncode从低到高依次调用接口。低于渠道上最高版本的包，不允许上传。


情景二：
手里有一个包，目的很明确，就是用来创建渠道，或把这个包更新更新到某个渠道上
1.新增渠道：直接请求addchannel接口，如果渠道存在，则提示渠道已存在，如果渠道不存在，则创建一个渠道
2.更新版本：先通过chennellist接口获取该渠道的subchid，subchid和文件作为参数请求updatechannel来更新版本
这种情形，请参考demo2.php
注1.渠道存在，不会自动更新，渠道不存在，不会自动创建
注2.更新时可能会有交互，需二次确认或需要增加或重置一些参数
